АНАЛИЗ И ОБЪЯСНЕНИЕ РЕЗУЛЬТАТОВ

ПОЧЕМУ АСИНХРОННОЕ СКАЧИВАНИЕ БЫСТРЕЕ?

1. СИНХРОННЫЙ ПОДХОД (Последовательно)
   1. Отправляем запрос на сервер №1
   2. ЖДЕМ ответа от сервера №1 (программа заморожена)
   3. Получили ответ, отправляем запрос на сервер №2
   4. ЖДЕМ ответа от сервера №2 (программа заморожена)
   5. И так далее...
   ИТОГО ВРЕМЯ: примерно 8 × (время одного запроса) = ~10.4 сек

2. АСИНХРОННЫЙ ПОДХОД (Параллельно)
   1. Отправляем запрос на сервер №1
   2. Тут же отправляем запрос на сервер №2
   3. Тут же отправляем запрос на сервер №3
   4. ... отправляем запросы на ВСЕ серверы
   5. ЖДЕМ ответа от ВСЕХ серверов ОДНОВРЕМЕННО
   6. Когда самый медленный сервер ответит - готово!
   ИТОГО ВРЕМЯ: примерно 1 × (время самого медленного запроса) = ~2.3 сек

3. КЛЮЧЕВОЕ РАЗЛИЧИЕ
   СИНХРОННО: Получаем ответ №1 → ПОТОМ отправляем запрос №2
             Фронт-процесс ЖДЕТ (блокирущее ожидание)

   АСИНХРОННО: Отправляем ВСЕ запросы → ПОТОМ ждем ответов
              Пока ждем от сервера №1, мы уже общаемся с другими
              Программа НИКОГДА не замирает

4. АНАЛОГИЯ ИЗ ЖИЗНИ
   СИНХРОННО:
   Кассир обслуживает клиентов очередью:
   - Обслужил клиента №1 (5 минут) → тот ушел
   - Обслужил клиента №2 (3 минуты) → тот ушел
   - Обслужил клиента №3 (7 минут) → тот ушел
   Итого: 5 + 3 + 7 = 15 минут

   АСИНХРОННО:
   8 кассиров работают параллельно, каждый со своим клиентом:
   - Кассир №1 обслуживает 5 минут
   - Кассир №2 обслуживает 3 минуты
   - Кассир №3 обслуживает 7 минут
   Итого: max(5, 3, 7) = 7 минут (время самого долгого!)

5. ФОРМУЛА УСКОРЕНИЯ
   Ускорение = Время синхронного / Время асинхронного
   Ускорение = 10.40 / 2.30 = 4.51x

   Это означает, что асинхронный способ в 4.5 раз БЫСТРЕЕ!

6. КОГДА ИСПОЛЬЗОВАТЬ АСИНХРОННОСТЬ?
   - Много сетевых запросов (API, базы данных, скачивание файлов)
   - I/O операции (чтение с диска, работа с интернетом)
   - Когда много операций, которые ждут ответов

   - НЕ нужна асинхронность для тяжелых вычислений (CPU bound)
   - НЕ нужна асинхронность для простых операций

ВЫВОД: Асинхронное программирование экономит ВРЕМЯ ОЖИДАНИЯ!
